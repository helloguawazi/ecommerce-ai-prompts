# 每日同步操作说明

## 日常更新流程（每次写完内容后）

在本项目文件夹里打开终端，执行下面三条命令：

```
git add .
git commit -m "新增：文章标题"
git push
```

就这三步，内容就同步到 GitHub 了。

## 写新文章的规范

1. 新文章放在 `prompts/` 目录，文件名用「编号-关键词」，例如 `002-服装详情页-模特展示.md`
2. 效果图放在 `images/` 目录，图片文件名用英文关键词，例如 `002-clothing-detail-model.jpg`
   - 图片文件名带关键词，有利于图片被搜索引擎收录
3. 每篇文章记得在 `README.md` 的目录里加一行链接
4. 文章末尾保留「一键生成」的独立站链接

## 重要提醒（避免拖累独立站）

- GitHub 上的内容不要和独立站文章一模一样。独立站放完整版，GitHub 放精简版或换个角度写。
- 每篇都要有真实价值（提示词能用、有说明、有效果图），不要纯堆链接。
- 独立站链接自然植入即可，不要每段都塞。

## 独立站地址替换

搭好独立站后，把 `README.md` 和各篇文章里的 `https://你的独立站地址` 替换成真实地址。
